No BS Guide to GitHub

What is GitHub?

GitHub is a platform where developers store, manage, and collaborate on code. Think of it as Google Drive for code, with version control powered by Git.

Common Git Terms (No Fluff)

  • git init: Start tracking a project with Git (like setting up a blank notebook for your code).
  • git add .: Stage all changes in your code (select files to save).
  • git commit -m "message": Save your changes locally with a description.
  • git push: Upload your changes to GitHub.
  • git pull: Download the latest changes from GitHub.
  • git merge: Combine changes from one branch into another.
  • git rebase: Reapply commits from one branch on top of another (tidy history).
  • Branch: A parallel version of your project (default: main).
  • Repository (repo): A folder containing your project and its history.

Step-by-Step Instructions to Use GitHub

1. Setting Up

  1. Install Git: Download from git-scm.com.
  2. Create a GitHub Account: Sign up at github.com.

2. Starting a Project

cd path/to/your/project
git init

3. Adding Files and Committing Changes

git add .
git commit -m "Initial commit"

4. Linking to GitHub

  1. Create a repository on GitHub.
  2. Link your local project: 
    git remote add origin https://github.com/your-username/your-repo-name.git

5. Pushing Your Code

git push -u origin main

6. Making Changes

git add .
git commit -m "Describe your change"
git push

7. Pulling Changes

git pull origin main

8. Working with Branches

  • Create a branch: 
    git checkout -b feature-name
  • Merge into main: 
    git checkout main
git merge feature-name
git push

9. Advanced Commands

  • Rebase: 
    git rebase main
  • Resolve Merge Conflicts: Fix conflicted files, then: 
    git add .
git commit

10. Cloning a Repo

git clone https://github.com/user/repo-name.git

11. Contributing to Open Source

  1. Fork a repository on GitHub.
  2. Clone it: 
    git clone https://github.com/your-username/forked-repo.git
  3. Make changes, commit, and push.
  4. Open a Pull Request (PR) on GitHub.

Golden Rules

  • Commit often: Don't wait for big changes.
  • Write clear commit messages: Summarize what changed.
  • Pull before you push: Always get the latest changes.
